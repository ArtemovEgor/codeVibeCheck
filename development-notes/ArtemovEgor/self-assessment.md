## Описание вашей работы

**Что реализовано:**
В рамках проекта была проведена большая работа по интеграции и разработке компонента AI-интервью и чата:

- Полностью реализовал компонент AI-интервьюера (frontend и backend), который динамически улавливает сильные и слабые стороны пользователя.
- Написал свою систему парсинга ответов ИИ, реализовал механику ограничения диалога на основе количества сообщений.
- Добавил бэкенд ИИ-чата на основе Node.js с API для сохранения и экспорта результатов чатов в маркдаун.
- Провел инфраструктурную настройку бэкенда: поднял VPS (Digitalocean), настроил nginx с SSL-шифрованием, написал `Dockerfile` и автоматизировал развертывание проекта через Github Actions.
- Настроил кастомный роутинг (Singleton-паттерн) с возможностью обработки query-параметров, реализовал поддержку модалок, обработку клиентских ошибок и настроил страницу 404.
- Полностью реализовал фронтенд для авторизации и регистрации: модальные окна на главной странице, UI-состояния.
- Написал слой API (API Layer) для авторизации, внедрив переключение между моковым и реальным бэкендом.
- Настроил на клиенте логику сессии: корректную отработку ошибки `401 Unauthorized` и принудительный логаут при сгорании токенов.

**Сложности и технологии:**

- **Технологии**: Vanilla TypeScript с Vite для фронтенда, Node.js для бэкенда. Docker, Nginx, Github Actions для развертывания.
- **Главные сложности**: Много времени заняла "борьба с промптами" и создание собственного надежного парсера ответов от LLM, чтобы управлять завершением диалога. Также сложным был этап деплоя, особенно с учетом необходимости преодолеть mixed content при обращении с фронтенда на внешний бэкенд через VPS.

## 2 личных Feature Component (для презентации)

1. **AI Chat UI & LLM Logic (Complex Component, +25 баллов):**
   Полная реализация чат-платформы, в которой ИИ самостоятельно определяет формат ведения диалога, а по его окончании возвращает форматированный результат для парсинга на фронте. Здесь задействованы нативные запросы и стриминг данных.
2. **AI Backend Service & Infrastructure (Custom Backend + Docker, +40 баллов):**
   Самостоятельно созданный бэкенд-сервис для обработки ИИ, который был полностью контейнеризован при помощи Docker и настроен на автоматическую сборку (CI/CD Github Actions) на выделенном виртуальном сервере, обеспечивающий стабильную работу API для всего дашборда команды.
